# Contributing

Thank you for contributing to Torchify.

## Branch Naming

- feat/<feature-name>
- fix/<bug-name>
- docs/<documentation-name>
- chore/<task-name>

## Commit Messages

Follow Conventional Commits.

Examples:

- feat: add flashlight interaction
- fix: resolve modal positioning
- docs: update README

## Pull Requests

Before opening a pull request:

- Link the related issue.
- Keep changes focused.
- Verify functionality.
- Ensure no unnecessary files are committed.

## Code Style

- Use semantic HTML.
- Keep CSS organized.
- Write readable JavaScript.
